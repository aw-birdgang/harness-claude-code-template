# Common coding style

- Prefer small, reviewable changes; match existing project patterns.
- Do not commit secrets, personal absolute paths, or machine-specific state.
- When merging ideas from a larger reference harness, follow `HARNESS_NODE_CONDENSE_STRATEGY.txt` to avoid duplicate trees.
- For harness evolution (rules/skills/agents backlog), align with `MONOREPO_STACK_HARNESS_GAP_ANALYSIS.txt` so target-stack work stays harness-side, not product scaffolding in this repo.
